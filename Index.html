# Caring First Website Static website for Caring First — ready for GitHub Pages deployment. ## Files - `index.html`, `about.html`, `services.html`, `careers.html`, `faq.html`, `contact.html` - `styles.css`, `script.js` - `assets/` (logo and images) - `.nojekyll` (ensures GitHub Pages serves files as-is) - `README.md` (this file) ## Deploy on GitHub Pages (Steps) 1. Create a repo named `caring-first-website` (Public). 2. Upload all files/folders from this directory to the repo root. 3. In the repo, go to **Settings › Pages**. 4. Under **Build and deployment**, set **Branch** to `main` and **Folder** to `/ (root)`, then **Save**. 5. Wait 1–2 minutes. Your site will appear at: `https://.github.io/caring-first-website/` ## HubSpot Forms (Contact & Careers) - In HubSpot: **Marketing › Lead Capture › Forms › Create Embedded form**. - Add your fields (Name, Email, Phone, Message, File Upload for Contact; Name, Email, Phone, Resume Upload for Careers). - Click **Publish**, copy the **Embed code**. Paste your embed code into: - `contact.html` where you see: `` - `careers.html` where you see: `` Example placement: ```html ``` ## Custom Domain (Optional) - In repo **Settings › Pages**, click **Custom domain** to connect a domain you own. - Add a `CNAME` file to the repo root with your domain to persist the setting. ## Updating the Site - Edit files locally and upload again, or connect the repo to your computer and push commits.
